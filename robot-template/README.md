# Шаблон для создания нового робота

Следуя этой инструкции, вы разработаете собственного робота, который затем можно загрузить в программу **Robots** через меню *Робот → Загрузить из JAR...*.

## Краткая справка

Приложение моделирует движение робота к цели, задаваемой щелчком мыши. Вы можете изменить две вещи:
- **Логику движения** — как робот перемещается к цели.
- **Внешний вид** — как робот отрисовывается на игровом поле.

Для этого нужно реализовать три интерфейса, после чего правильно упаковать всё в JAR-файл.

## Шаг 1. Подготовка проекта

Создайте обычный Java-проект. Интерфейсы, которые требуется реализовать, уже присутствуют в основном приложении. Чтобы ваш код компилировался, достаточно временно скопировать к себе файлы из пакета `robot` (`RobotBehavior.java`, `RobotRenderer.java`, `RobotFactory.java`) и класс `RobotState.java`, **не включая их в итоговый JAR**. 

## Шаг 2. Реализация интерфейсов

### 2.1. Поведение робота — интерфейс `RobotBehavior`

Метод этого интерфейса вызывается на каждом шаге симуляции (каждые 10 миллисекунд). Он получает текущие координаты и направление робота, координаты цели и промежуток времени `dt`, а должен вернуть новое состояние в виде объекта `RobotState`, содержащего новые `x`, `y` и направление в радианах.

Вы можете реализовать любую логику. Все вычисления делаются в радианах!

### 2.2. Отрисовка робота — интерфейс `RobotRenderer`

Метод отрисовки получает графический контекст, а также координаты и направление.

### 2.3. Фабрика — интерфейс `RobotFactory`

Фабрика отвечает за создание экземпляров поведения и рендерера. Методы `createBehavior()` и `createRenderer()` возвращают ваши реализации соответствующих интерфейсов.

## Шаг 3. Регистрация фабрики через ServiceLoader

В корне вашего исходного кода создайте каталог `META-INF/services`. Внутри него создайте текстовый файл с именем `robot.RobotFactory`. В этом файле укажите полное имя вашего класса-фабрики, например `my.robot.MyRobotFactory`.

## Шаг 4. Сборка JAR-файла

Скомпилируйте код и упакуйте его в JAR. Внутри архива должна быть следующая структура:
- `META-INF/services/robot.RobotFactory`.
- Скомпилированные классы вашей реализации (`my/robot/MyBehavior.class`, `my/robot/MyRenderer.class`, `my/robot/MyRobotFactory.class`).


## Шаг 5. Проверка работы

1. Запустите приложение Robots.
2. В меню выберите **Робот → Загрузить из JAR...** и укажите ваш файл.
3. При успешной загрузке в логе появится сообщение «Загружен робот из …».
4. Щёлкните по игровому полю — робот начнёт двигаться согласно вашей логике и рисоваться вашим рендерером.
5. Вернуться к стандартному роботу можно через пункт **Робот → Стандартный робот**.
