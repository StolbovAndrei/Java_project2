# Robots

Учебный проект для изучения принципов объектно-ориентированного дизайна, паттерна MDI (Multi-Document Interface) и разработки на Java.

## История проекта

Исходный код получен из репозитория [alklepin/Robots](https://github.com/alklepin/Robots). Базовая версия содержала многооконное приложение с графическим интерфейсом, окном протоколирования и симуляцией движения робота. Однако в архитектуре имелись ошибки проектирования и недостатки, которые были исправлены и расширены в процессе выполнения учебных задач.

## Выполненные задачи

### 1. Знакомство с MDI-приложением
- Добавлен пункт меню **«Выход»**, позволяющий корректно завершить приложение.
- Обработка выхода вынесена в единый метод, вызывающий диалог подтверждения с помощью `JOptionPane`.
- Все стандартные тексты кнопок и подсказок интерфейса переведены на русский язык через установку соответствующих значений в `UIManager` (файлы локализации `ComponentsMenu_ru.properties` и `ComponentsMenu_en_US.properties`).

### 2. Управление состоянием окон
- При выходе из приложения сохраняется положение и состояние главного окна и всех внутренних окон.
- При запуске состояние восстанавливается из файла `robots_config.properties`, расположенного в домашнем каталоге пользователя.
- Реализовано в классе `ConfigManager`.

### 3. Знакомство с MVC
- Модель робота (`RobotModel`) выделена как наблюдаемый объект, уведомляющий зарегистрированных слушателей (`RobotModelListener`) об изменении состояния.
- Добавлено новое окно **«Информация о роботе»** (`RobotCoordinatesWindow`), которое отображает текущие координаты, направление, расстояние и угол до цели. Окно подписывается на обновления модели и автоматически перерисовывается.
- Исправлена ошибка логики движения: ранее при определённом взаимном расположении робот уходил в противоположном направлении. Теперь реализован корректный расчёт угловой скорости (класс `DefaultRobotBehavior`).

### 4. Загрузка внешних роботов из JAR
- Реализован механизм динамической загрузки логики и визуализации робота из внешнего JAR-файла.
- Используются `URLClassLoader` и `ServiceLoader` для поиска и инстанцирования `RobotFactory`.
- Добавлены пункты меню **«Загрузить из JAR...»** и **«Стандартный робот»**.
- Перед загрузкой проверяется, что файл действительно является JAR-архивом (по сигнатуре PK\x03\x04).
- Интерфейсы, которые должен реализовать внешний робот: `RobotBehavior`, `RobotRenderer` и `RobotFactory`.
- Подробное описание создания такого робота вынесено в отдельный [шаблон](./robot-template/README.md).

## Локализация

Приложение полностью поддерживает два языка: русский и английский. Язык переключается в меню **Язык** через `LanguageManager` и интерфейс `LocalizableComponent`. Все надписи меню, заголовки окон и системные строки подгружаются из ресурсных файлов.
